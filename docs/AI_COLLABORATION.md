# AI 协作说明

本项目开发过程中使用 Codex 和 Claude 辅助进行代码实现、方案推敲、文档整理、检查验证和版本发布。公开仓库只保留协作方式层面的说明，不公开具体项目设计对话、任务看板、handoff 记录、内部路线规划或未发布创意。

## 协作方式

- 人类负责人确定需求、产品取舍、公开范围和最终发布。
- Codex 主要辅助本地代码实现、工程化整理、测试检查、Git/GitHub 发布和文档更新。
- Claude 主要辅助方案审查、分析口径推敲、文案组织和风险提示。
- 关键改动以可运行源码、测试结果和可追溯提交为准，而不是仅停留在对话结论。

## 公开边界

- 公开：源码、示例配置、部署说明、基础测试、适合展示的 README 和开发快照。
- 不公开：真实业务数据、采集结果、密钥、数据库配置、账号信息、内部任务看板、详细设计讨论和未来规划。
- 涉及联网采集、写库、任务触发或 Agent 操作的功能，应保留人工确认和本地配置边界。

这个文件用于说明项目如何使用 AI 工具提升工程效率，不代表公开完整的内部协作记录。
